1. Overview

Jud Dashboard is an internal management platform built for Jud Agency, a digital marketing agency in Saudi Arabia. The platform allows account managers to monitor and manage Google Ads campaigns for multiple clients from a single unified dashboard.

2. Purpose of Google Ads API Usage

We use the Google Ads API to:
  • Fetch campaign performance data (impressions, clicks, cost, conversions)
  • Retrieve campaign reports for specific date ranges
  • Display real-time metrics to account managers
  • Monitor ROAS (Return on Ad Spend) across client accounts
  • Manage multiple client accounts under one MCC account

3. API Endpoints Used

EndpointPurpose
GoogleAdsService.Search (GAQL)Fetch campaigns and performance metrics
customers:listAccessibleCustomersList accessible client accounts
customers/{id}/googleAds:searchQuery campaign reports by date range

4. Data Flow

Step 1: Account manager connects a client's Google Ads account via OAuth2 credentials
Step 2: Dashboard fetches campaign data using GAQL queries
Step 3: Data is cached in Supabase database to minimize API calls
Step 4: Account managers view metrics in the campaigns dashboard

6. Access Control

The dashboard is accessible to internal users only (Jud Agency employees and account managers). Each user authenticates via Google OAuth before accessing the platform. Client data is isolated per store/account and only visible to authorized managers.

8. API Usage Compliance

  • We do not resell or redistribute Google Ads data
  • Data is used solely for internal campaign management purposes
  • We comply with Google Ads API Terms of Service
  • API calls are rate-limited and cached to minimize quota usage
  • No automated bidding or campaign creation without human approval
